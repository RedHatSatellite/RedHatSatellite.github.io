Welcome to Red Hat Satellite

This organization exists to provide a location to curate scripts/projects that Red Hat employees, customers, and community members have written for usage with Red Hat Satellite. We would like to leverage this organization in the following manner

  • Provide a clearinghouse of well maintained tools known to work with a supported version of Satellite.
  • Allow users of Satellite to showcase their work and develop their brand.
  • Provide a process for content to eventually become part of the Foreman upstream (and eventually a proper part of the product).

Contributing

Want to contribute? We don't ask for much. Mainly that:

  • You actively maintain your code/projects and are responsive to issues/PRs
  • You denote which version of Satellite your code/projects work against
  • You provide documentation on the proper usage of your code/projects

Still interested? Reach out to one of the maintainers below so that we can add your project. Our onboarding process works like such

For repos not currently hosted on Github


  • One of the maintainers will create a new repo under the RedHatSatellite org, with your original repo being used as the source for it
  • We'll add you as an admin of the repo
  • We'll invite you to join the RedHatSatellite org
  • Reach out to external contributors to inform them of the new location
  • Bob -> Uncle

For repos currently hosted on Github


Note: this process is based on the Standard Github Transfer process, and will allow preservation of commit history, open issues, PRs, etc.
  • Coordinate the repository transfer with one of the maintainers below. (You'll transfer the repo to them. Then they will transfer it into the Satellite org)
  • We'll add you as an admin of the repo
  • We'll invite you to join the RedHatSatellite org
  • Reach out to external contributors to inform them of the new location
  • Bob -> Uncle
Also, take a look at the example repo for a template for how to structure your README
